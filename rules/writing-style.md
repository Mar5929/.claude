## Writing Style — Avoid AI-Generated Language Patterns

When producing any written content (documents, emails, reports, presentations, or code comments), avoid the following:

### Prohibited Punctuation

- **No em dashes** (`—`). Use a colon, semicolon, or restructure the sentence.

### Prohibited Phrases

Never use these phrases or close variants:

- "delve into"
- "it's worth noting"
- "it is worth noting"
- "this is a testament to"
- "it is important to note"
- "it is worth mentioning"
- "in the realm of"
- "leveraging" (when meaning "using" — use "using" instead)
- "utilize" / "utilizing" (use "use" / "using" instead)
- "in today's fast-paced world"
- "at the end of the day"
- "it goes without saying"
- "needless to say"
- "touch base"
- "circle back"
- "game-changer"
- "robust" (when describing software features generically)
- "seamlessly"
- "synergy" / "synergize"
- "holistic approach"
- "moving forward"
- "going forward"
- "take it to the next level"
- "dive deep" / "dive into"

### Style Guidelines

- Active voice throughout
- Short, direct sentences
- Tables over prose where possible
- Numbers, not vague estimates
- No hedging language ("arguably", "perhaps", "it could be said that")
- Lead with the point, not the context

### When These Rules Apply

Every document, email, presentation, or written artifact produced for any project. This is a global standard.
